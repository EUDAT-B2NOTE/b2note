Login page ^top

Welcome to the B2NOTE annotation service.

B2NOTE is currently designed as a crowdsourcing service which implies that all the annotations are accessible publicly. To guarantee on one side your privacy and on the other side to keep track of the origin of annotation records, you must first register to create an annotator profile with a pseudonym which will be used to publish your annotation.

This page is the entry point to access the service and allows you either register as first time user (2) or log in directly if you already created an account (1). In case you forgot your password, you can re-initialize it by clicking on the link “Forgot password” (3)

Registration^top

Fill in the required information to create an annotator profile and do not forget to enter the captcha before submitting.


If you forgot to fill-in some information or provided an invalid email address, a red error message will appear above the field to correct.

Upon successful registration you should be redirected to the Login page.

Main page ^top

Create and sum-up annotations.

Why annotating? Using B2NOTE you can add and manage extra information (“annotations”) about data files hosted by a EUDAT service (a B2SHARE training instance during the pilot phase). Then use the annotations for finding files of interest and identifying new user-specific datasets.

This is the B2NOTE service main page. It contains the following elements:

  1. Navigation bar: access different functionalities of the B2NOTE service.
  2. Main panel: create an annotation.
  3. “All my annotations”: visualise a summary of your annotations and access more detailed information.
  4. “All annotations about this file”: visualise a summary of the annotations about the current file and access more detailed information.
  5. Feedbacks button: assess the service, request a new feature or report a bug.
Navigation bar

  • "User account: access your annotator profile information.

  • Search: find files using annotations and export the resulting datasets.

  • Export: export annotations in JSON-LD (linked-data) format.

  • Logout and close: log out of and close the B2NOTE service, terminates the current user session.

  • Help: access the help section for the page you are visiting.

Navigation bar

At the top of other pages additional shortcut links allow quick navigation to previous pages or straight back to the B2NOTE console main page.

  • Previous: previously accessed page.

  • Home: the B2NOTE console main page.

Main panel

Create annotations.

The main panel is divided-up into 3 tabs. Each tab contains a simple form for creating an annotation of one of the 3 types defined in B2NOTE.

  1. Semantic tag: create an annotation that uses a term defined in a controlled vocabulary as a tag.

  2. Free-text keyword: create an annotation that uses your own keyword as a plain-text tag.

  3. Comment: create an annotation containing a longer piece of text.

Creating a semantic tag annotation

The semantic tag annotations use a term defined in a controlled vocabulary as a tag. This type of annotations thus benefits from curated and uniquely identified linked-data material.

Clicking the text-area then typing-in the first letters of the term you want triggers the autocomplete functionality, which provides a list of suggestions.

From the list of suggestions, you can then select your term of interest amongst the extensive B2NOTE index.

After selecting one of the suggestions, click the “Create” button to create the annotation.

Creating a free-text keyword annotation

Free-text keyword annotations are designed for cases when the exact term you want is missing from the “Semantic tag” autocomplete list of suggestions and you want a plain-text tag to be used in the annotation.

Annotations of this type contain plain text information as opposed to linked-data, thus not allowing the full range of benefits provided by the “Semantic tag” type.

After typing the text for your custom keyword in the text-area, click the “Create” button to create the annotation.

Creating a comment annotation

Comment annotations are designed for annotating using longer pieces of text. That is, possibly more verbose text not intended for the purpose of tagging a document, such as commenting or leaving some evaluating or general quality feedback.

Annotations of this type also, like the “free-text keyword” type, contain plain text information as opposed to linked-data, thus not allowing the full range of benefits provided by the Semantic tag” type.

After typing-in the text of the comment, click the "Create" button to create the annotation.

After selecting a semantic tag or typing-in a free-text keyword or comment text, click the “Create” button to create the annotation.

Some conditions prevent annotations from being created.

  1. Duplicate: it is not possible to create an annotation that would be a duplicate of an existing annotation. So a semantic tag or free-text tag annotation can not use the same tag about the same file as another annotation. When attempting to do so, an alert will be displayed informing you that this is the case.

  2. Semantic equivalent: attempting to create a free-text keyword annotation using a label identical to an ontology term referenced by B2NOTE, triggers an alert asking you to confirm the type of annotation you want to create:

    • Semantic: select a semantic tag instead of your free-text keyword.

    • Keyword: confirm creating an annotation using your free-text keyword.

    • Cancel: cancel the annotation creation.

  3. Long keyword: attempting to create a free-text keyword annotation with a tag label longer than 60 characters will trigger an alert asking to confirm the type of annotation you want to create:

    • Keyword: for an annotation of the Free-text Keyword type.

    • Comment: for an annotation of the Comment type.

    • Cancel: to cancel the annotation creation.

My annotations page ^top

Review and manage your existing annotations.

Access this page from the B2NOTE console main page by clicking the “All my annotations” link.

  1. Summary table: showing how many annotations of each type were created by you and linking to the corresponding detailed information tables.

  2. Detailed information table: on this page there are 3 such tables, one for each type of annotation. They show the labels of corresponding annotations that were created by you, link to individual annotation data and links for editing or deleting each annotation.

Summary table

Referencing the number of annotations of each type that were created by you.

Click each link or numbered badge to a table containing more detailed information about annotations of the corresponding type (see below).

Detailed information table

There is one such table per annotation type (Semantic tag, Free-text keyword and Comment). Each row of the table features:

  1. Label of the annotation: click to access detailed information about the tag.

  2. Numbered badge: reporting how many of your annotations use the same tag. Click to access the corresponding list of files and detailed information about the tag. There are no numbered badges in the Comment annotation table since these annotations are by definition unique.

  3. Management options:

    • Pen icon: edit the corresponding annotation.

    • Bin icon: delete the corresponding annotation.

All annotations page ^top

Review existing annotations about a file.

Access this page from the B2NOTE console main page by clicking the “All annotations about this file” link.

  1. Summary table: showing how many annotations of each type were created about the current file and linking to the corresponding detailed information tables.

  2. Detailed information table: on this page there are 3 such tables, one for each type of annotation. They show the labels of corresponding annotations, link to individual annotation data and links for editing or deleting each annotation.

Summary table

Referencing the number of annotations of each type that were created, either by other users (blue badges) or yourself (orange badges).

Click each link or numbered badge to a table containing more detailed information about annotations of the corresponding type (see below).

Detailed information table

There is one such table per annotation type (Semantic tag, Free-text keyword and Comment). Each row of the table features:

  1. Label of the annotation: click to access detailed information about the tag.

  2. Numbered badge: reporting how many annotations use the same tag. Click to access the corresponding list of files and detailed information about the tag. There are no badges in the Comment annotation table.

    • Blue badge: annotations created by other users.

    • Orange badge: annotations created by you.

  3. Management options: these actions are disabled (pale grey icons) for annotations created by other users.

    • Pen icon: edit the corresponding annotation.
    • Bin icon: delete the corresponding annotation.

Annotation summary page ^top

Visualise the content of an individual annotation.

Access this page from detailed summary tables of the “My Annotations” or "All annotations” pages.

  1. Annotation content: information about the annotation label. For Semantic tags this is information about the original ontology term, including its URI as an hyperlink for direct access. Click the topmost chevron icon to fold or unfold the table.

  2. List of files: direct links to the files that were annotated using the same annotation tag.

Edit annotation page ^top

Change an annotation.

Access this page by clicking one of the pen icons (circled in red above) in tables of the “My Annotations” or "All annotations” pages.

Edit menu

Allows the tag of an individual annotation to be modified.

Type inside the textarea to select a different semantic tag from the list of suggestions (in case of editing a semantic tag) or modify the text of free-text annotations (i.e. keyword or comment types).

  • Save: enforce the modified tag as the new tag.

  • Cancel avoid changing the original tag.

The process of editing the annotation tag is very similar to that of creating the tag for a new annotation from the main page interface. The same restrictions and alerts apply. In particular it is not allowed to generate a duplicate (same tag about the same file) annotation.

Delete annotation page ^top

Delete an annotation.

Access this page by clicking one of the bin icons (circled in red above) in tables of the “My Annotations” or "All annotations” pages.

Delete menu

Allows deleting an individual annotation. This option is only available for the annotations that you have created.

  • (red) Delete: confirm that the annotation is to be deleted.

  • Cancel: avoid deleting the annotation.

Search page ^top

Use annotations to search for files.

Access this page by clicking the magnifying glass icon in the navigation bar from other pages of the B2NOTE console.

The Search functionality is designed for identifying files annotated with some particular tag information.

Define your search query by filling-up the form of the Search menu.

  1. Search selector: define a term to search for.

  2. Plus sign button: add new terms to the search query.

  3. Magnifying glass button: launch the search.

Search selector

First select a particular annotation type.

Then type in the textarea to select (in case of a semantic tag), or define a particular tag label to be searched on.

In the case of semantic tag annotations, a tick box underneath the textarea allows you to require that the synonyms referenced for the term in the original ontology be included, so that files annotated with tags matching the synonyms also get reported in the search results.

Select the Comment annotation type for including in the search results those files for which at least one of their annotation is of the “Comment” type.

Plus sign button

Click the “plus sign” button (circled in red above) to add new conditions to your search query by adding new field to the search form.

The additional search fields are identical to the initial form with the addition of a selector for the logical relation to apply between search conditions.

  • AND: both conditions true.

  • OR: at least one condition true.

  • NOT: corresponding condition not true.

  • XOR: strictly one or the other condition true.

Magnifying glass button

Click the “magnifying glass” button (circled in red above) to launch the search according to the parameters thus defined.

You obtain the list of files with annotations matching your query displayed in the search results page (see below).

Search results ^top

Identify files with annotations matching a search query.

Access this page by clicking the magnifying glass button after filling up the form of the Search page for submitting a search query.

  1. Search query: formulated in plain text.

  2. New search: define and execute a new search.

  3. Export results: for selecting within the search results those particular files about which annotation information should be exported.

  4. Exact match: list the files that were retrieved on the exact match of their annotation content with the search query conditions.

  5. Related match (synonyms): list the files that were retrieved because their annotation label matched with a synonym of a particular semantic tag (see Search page) included in the search query. If synonyms were listed for the original ontology class, then these should appear in the above query formulated as plain text.

Export search results page ^top

Define datasets of files from search results and export their annotation information.

Access this page by clicking the “Export results” button of the Search results page.

The layout of this page is similar to that of the “search results page”, with additional tick boxes (1) for selecting/un-selecting the listed files about which annotation information should be included for exporting.

Once file selection is completed, clicking the “Generate JSON export” button (2) loads a page displaying the corresponding list of annotations in JSON-LD format and making it available for download.

Export page^top

Download annotations in JSON-LD format.

Access this page by clicking the “Export” icon in the navigation bar from other pages of the B2NOTE console.

  • All my annotations: display the information of all your annotations in JSON-LD (linked-data) format and ready to download.

  • All annotations about the file: display the information of annotations about the current file from all users in JSON-LD (linked-data) format, and ready to download.

Click one of the two buttons to a page with the following layout:

  • Annotations information: JSON-LD (linked-data) format.

  • Download JSON-LD: download the text file of the displayed JSON-LD.

User account page ^top

Visualise your annotator profile information.

Access this page by clicking the User account icon in the navigation bar page displaying the data filled-in during registration (user-specific information).

This page displays your annotator profile information as filled-in during your registration to the service, with the obvious exception of the password field.

Feedback page ^top

Provide your assessment, feature request or bug report about B2NOTE service.

Access this page by clicking the “Let us know what you think” button at the bottom of the B2NOTE console main page.

This page comprises a single panel divided-up in 3 tabs. Each tab contains a form for sending a different set of informations to the service maintainers.

  1. Assess the service: provide you ratings about the service and more specific comments.

  2. Request a new feature: provide the description of a feature or functionality that could improve the service.

  3. Report a bug: provide information describing a service disfunction that you identified.

Each form is to be completed and submitted independently.

Fill in the required information and do not forget to enter the captcha before submitting.
